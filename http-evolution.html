Evolution of HTTP

How HTTP changed from a simple document protocol into the foundation of the modern web.

HTTP/0.9

The earliest version of HTTP appeared in 1990 when Tim Berners-Lee created the first web technologies. HTTP/0.9 was extremely simple and could only request and return plain text documents.

HTTP/1.0

HTTP/1.0 was formally documented in 1996. It introduced headers, status codes, and content types, which made the protocol much more useful. However, every request still needed a new TCP connection, which created inefficiency as websites became more complex.

HTTP/1.1

In 1999, HTTP/1.1 improved performance by adding persistent connections, better caching, and chunked transfer encoding. It became the dominant version of HTTP for many years and is still widely used.

HTTPS

Because HTTP alone does not encrypt data, secure web communication required the use of Transport Layer Security (TLS). Combining HTTP with TLS created HTTPS, which protects users from eavesdropping and tampering.

HTTP/2

Standardized in 2015, HTTP/2 was designed to improve performance for modern websites with many page resources. It introduced multiplexing, binary framing, and header compression to reduce overhead and improve speed.

HTTP/3

Standardized in 2022, HTTP/3 moved away from TCP and instead uses QUIC over UDP. This helps reduce latency, avoid head-of-line blocking, and improve performance on mobile and unreliable networks.

Major HTTP Versions
Version Year Main Improvement
HTTP/0.9 1990 Basic document retrieval
HTTP/1.0 1996 Headers, status codes, content types
HTTP/1.1 1999 Persistent connections and better caching
HTTP/2 2015 Multiplexing and lower overhead
HTTP/3 2022 QUIC over UDP and lower latency
