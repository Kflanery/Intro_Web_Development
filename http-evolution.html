Evolution of HTTP

HTTP/0.9

The earliest version of HTTP appeared in 1990 when Tim Berners-Lee created the first web technologies. It could only request and return simple HTML documents.

HTTP/1.0

HTTP/1.0 introduced headers, status codes, and content types. However, each request required a new TCP connection, making it inefficient for growing websites.

HTTP/1.1

HTTP/1.1 improved performance with persistent connections, caching, and chunked transfer encoding. It became the long-term standard for the web.

HTTPS

HTTPS adds encryption using TLS, securing data between the browser and server and preventing interception or tampering.

HTTP/2

HTTP/2 introduced multiplexing, header compression, and binary framing to improve speed and reduce latency.

HTTP/3

HTTP/3 uses QUIC over UDP instead of TCP, reducing latency and improving performance on unreliable networks.

Major HTTP Versions
Version Year Main Improvement
HTTP/0.9 1990 Basic document retrieval
HTTP/1.0 1996 Headers, status codes, content types
HTTP/1.1 1999 Persistent connections and caching
HTTP/2 2015 Multiplexing and reduced overhead
HTTP/3 2022 QUIC over UDP and lower latency
