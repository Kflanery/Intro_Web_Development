1. HTTP

HTTP stands for Hypertext Transfer Protocol. It defines how web browsers and servers exchange information.

2. Client-Server Model

A client sends a request and a server returns a response. This is the basic communication model of the web.

3. URL

A URL identifies the address of a resource on the web, such as a webpage, image, or API endpoint.

4. Headers

Headers carry metadata in HTTP messages, such as content type, caching instructions, and browser details.

5. HTTPS

HTTPS is HTTP secured with TLS encryption, helping protect web traffic from interception and tampering.

6. Multiplexing

Multiplexing in HTTP/2 allows multiple requests and responses to travel at the same time over one connection.

7. QUIC

QUIC is the transport protocol used by HTTP/3. It is built on UDP and is designed to reduce delay and improve reliability.
